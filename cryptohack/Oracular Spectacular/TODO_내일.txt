== Oracular Spectacular - 내일 할 일 ==

[현재 상태]
- 8가지 전략 시도, 서버 15회+, 로컬 20회 → 전부 실패
- 근본 원인: per-byte 94% × 32바이트 = 전체 14% 성공률 + cascading error
- solve.py가 adaptive top-2 + 500회 자동재시도 버전으로 수정됨 (아직 미실행)
- local_test.py로 로컬 검증 가능 (0.1초/시도, 네트워크 불필요)

[내일 순서]

1. 로컬 성공률 측정
   python local_test.py 100 370
   → 현재 알고리즘의 실제 성공률 확인

2. 성공률 기반 판단
   - 1% 이상이면 → solve.py 서버 실행 (500회 자동재시도, 병렬 가능)
   - 1% 미만이면 → 전략 개선 필요

3. 아직 안 해본 전략들 (성공률 낮을 경우)
   a) 2차 CT 검증: encrypt 2번 → 1차로 복구, 2차로 바이트별 검증 → 오류 찾아 교체
   b) Sequential Halving: best-arm identification 최적 알고리즘
   c) 바이트 가중 budget: 초반 바이트(15,14)에 더 많은 쿼리 (cascade 피해 큼)
   d) backtracking: 검증 실패 시 2순위 후보로 교체 후 재진행

4. 서버 실행 (성공률 확보 후)
   python solve.py 500
   → 병렬: for i in 1 2 3 4 5; do python solve.py 100 > result_$i.txt 2>&1 & done
